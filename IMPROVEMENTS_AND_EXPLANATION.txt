================================================================================
  GAME OF THRONES QA SYSTEM — IMPROVEMENTS SUMMARY
================================================================================

RAG-based Q&A over all 5 A Song of Ice and Fire books using FAISS, CrossEncoder
reranking, and Claude Haiku for answer generation.

--------------------------------------------------------------------------------
PIPELINE
--------------------------------------------------------------------------------

  1. Chunking     Token-based (max_tokens=400, overlap=80) using MPNet tokenizer
  2. Embedding    all-mpnet-base-v2 (768d), saved to embeddings_0.npy
  3. Retrieval    FAISS IndexFlatIP + L2 norm (cosine), top_k=30
  4. Reranking    CrossEncoder ms-marco-MiniLM-L-6-v2, top-3 passages
  5. Expansion    ±4 chunk window around each top-3 hit (~27 chunks total)
  6. Answer       Claude Haiku (claude-haiku-4-5-20251001) with source attribution

--------------------------------------------------------------------------------
KEY CHANGES FROM ORIGINAL
--------------------------------------------------------------------------------

  1. all_chunks = metadata fix          Prevents NameError on kernel restart
  2. expand_window: 1 → 4              More context per query
  3. Top-3 reranked + windows           Cross-book answers captured
  4. Score threshold: 0.1 → 0.3        Filters weak extractions
  5. Score capped at 1.0               RoBERTa score artifact fix
  6. Claude Haiku replaces RoBERTa     Generative vs extractive; 200K context
  7. Token-based chunking              Prevents silent MPNet truncation
  8. Source attribution                Shows book/chunk for each answer
